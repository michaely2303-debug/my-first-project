# 금융 신용평점 예측 및 손실 위험 방어 머신러닝 프로젝트

## 1. 프로젝트 목적
- 금융/공공 데이터를 활용하여 단순 정확도(Accuracy) 향상이 아닌, **비즈니스 손실 위험 징후를 선제적으로 포착하고 방어**할 수 있는 분류 모델 구축

## 2. 주요 가설 및 파생 변수 설계 (Feature Engineering)
- **문제 인식**: 기존 회귀 모델 고집 시 데이터 잡음과 불균형으로 인해 실효성이 떨어진다고 판단.
- **가설**: 소득 대비 소비율이 높을수록 상환 여력이 낮아져 손실로 이어질 가능성이 큼.
- **해결 방안**: 데이터의 이상치를 단순 제거하지 않고 **'소득 대비 소비율'이라는 핵심 파생 변수를 설계**하여 회귀 문제를 분류 문제로 전환.

## 3. 모델 선정 및 성능 개선 과정 (Troubleshooting & Optimization)
- **초기 시도 (Random Forest)**
  - 널리 쓰이는 랜덤포레스트 적용 시, 심각한 클래스 불균형으로 인해 다수 클래스로 예측이 쏠려 **F1 Score 0.69**에 정체됨.
- **알고리즘 전환 (LightGBM)**
  - 이전 트리의 오차를 순차적으로 보완하는 부스팅 계열(LightGBM)이 소수 클래스의 오분류 학습에 더 유리하다고 판단하여 전환.
- **성과**: **F1 Score를 0.69에서 0.72로 개선** 및 모델 안정성 확보.

## 4. 최종 비즈니스 인사이트 및 느낀점 (Business Impact)
- 정형화된 알고리즘을 무조건 채택하기보다 **데이터의 특성(클래스 불균형)을 진단하고 적합한 기법을 찾아가는 과정**의 중요성 체득.
- 기술적 숙련도를 넘어 비즈니스 손실 리스크를 관리하는 실무형 데이터 분석가로 성장 중.
